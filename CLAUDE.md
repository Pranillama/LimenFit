# LimenFit — Claude session context

## Project

LimenFit: minimal, high-speed fitness tracking app (workout logging, AI-assisted progression, plan sharing, form analysis). Production: limen-fit.vercel.app.

**Stack:** Next.js 15 (App Router, RSC), TypeScript, Tailwind + shadcn/ui (new-york/neutral), Supabase (Postgres + auth + RLS + Storage), TanStack Query, Zod, react-hook-form, Zustand, Vitest.

## Layout conventions

- `app/(app)/<route>/` — App Router pages; server components by default, `'use client'` only when needed
- `app/api/<route>/` — route handlers; always `requireUser()` from `@/lib/api/auth`, wrap errors with `handleApiError`
- `features/<domain>/` — feature folders (`components/`, `hooks/`, `lib/`); barrel via `index.ts`
- `lib/<domain>/server.ts` — server-only helpers; top-level `assertServerOnly()`
- `lib/schemas/<name>.ts` — Zod schemas paired with DTO types
- `lib/supabase/types.ts` — generated by `pnpm dlx supabase gen types typescript --local`
- `supabase/migrations/` — SQL migrations, naming `<YYYYMMDDhhmmss>_name.sql`
- RLS-everywhere; weights stored as kg, heights as cm; UI converts at render

## Workflow (superpowers)

1. **brainstorming** before any feature work
2. **writing-plans** → save to `docs/superpowers/plans/YYYY-MM-DD-name.md` with bite-sized TDD tasks
3. **using-git-worktrees** → isolate via `EnterWorktree`
4. **subagent-driven-development** → fresh subagent per task + 2-stage review (spec, then quality)
5. **finishing-a-development-branch** → PR or merge cleanly

TDD for data-layer work (schemas, server helpers, API routes). UI primitives without logic are build-verify-commit.

## Local dev notes

- Migrations require `pnpm supabase start` + `pnpm db:reset` from the main repo path. Docker file-sharing must include `/Users/<you>` in Settings → Resources → File Sharing, otherwise containers fail to start.
- `.env.local` controls whether the app hits local Supabase (`http://127.0.0.1:54321`) or the linked remote — verify before debugging "table not found" errors.
- After adding a migration, if PostgREST returns `PGRST205`, restart the api container: `docker restart $(docker ps --filter "name=supabase_rest_limenfit" --format "{{.ID}}")` or stop/start supabase.
